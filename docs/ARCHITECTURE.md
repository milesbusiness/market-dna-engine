# Market DNA Engine — Architecture

## Overview

The Market DNA Engine is a statistical research platform that tags every trading day with 35+ objective conditions and stores them in DuckDB for SQL analysis. It is intentionally non-ML — all outputs are conditional probability from historical data.

## Data Model

Each row in `market_dna` table represents one trading day:

| Field | Type | Description |
|-------|------|-------------|
| `date` | DATE | Trading date |
| `symbol` | VARCHAR | e.g. `NIFTY 50_MINUTE` |
| `gap_type` | VARCHAR | `gap_up` / `gap_down` / `flat` |
| `gap_pct` | FLOAT | Gap size as % of previous close |
| `gap_filled` | BOOLEAN | Did price return to previous close? |
| `gap_continued` | BOOLEAN | Did price extend in gap direction? |
| `pdh_swept` | BOOLEAN | Did price breach Previous Day High? |
| `pdl_swept` | BOOLEAN | Did price breach Previous Day Low? |
| `pdh_sweep_stage` | INTEGER | 1–5 institutional sweep detection |
| `orb_direction` | VARCHAR | `long` / `short` — ORB breakout direction |
| `orb_success` | BOOLEAN | Did ORB reach 1× range target? |
| `orb_range` | FLOAT | High–Low of 9:15–9:45 window (points) |
| `vix` | FLOAT | India VIX at open |
| `vix_zone` | VARCHAR | `low` (<13) / `mid` (13–20) / `high` (>20) |
| `adx` | FLOAT | 14-period ADX |
| `regime` | VARCHAR | `choppy` / `ranging` / `mild_trend` / `trending` |
| `dte` | INTEGER | Days to NSE weekly expiry |
| `day_of_week` | INTEGER | 0=Mon … 4=Fri |
| `session_*_green` | BOOLEAN | Per-session green rate fields |

## Key Design Decisions

### 1. DuckDB over PostgreSQL
DuckDB is embedded, columnar, and handles 8 years of minute-bar aggregation in seconds without a running server. No connection management, no migrations — the `.duckdb` file is the database.

### 2. No ML, only conditional probability
Every edge in the system is of the form: *"When condition A and condition B, outcome C happened X% of the time across N observations."* This makes results auditable, explainable, and not subject to overfitting.

### 3. Minute bars → daily DNA
The engine reads raw minute OHLCV and derives daily properties (gap fill, PDH sweep, ORB range). This avoids storing derived data in the original TradeStore and keeps the DNA layer independent.

### 4. NSE expiry calendar computed programmatically
Weekly expiry dates are generated by rule (NIFTY = Thursday, BankNifty = Wednesday) rather than stored as data, ensuring correctness through holiday adjustments.

## Backtest Engine Architecture

```
POST /api/backtest
        │
        ▼
engine.py::run_backtest()
  ├── load_minute_data()         ← reads parquet from E:\TradeStore
  ├── Strategy selection
  │     ├── run_intraday_orb()   ← 8-filter ORB-30 strategy
  │     ├── run_swing_gapfill()  ← gap fade 0.2–0.8%
  │     └── run_position_trend() ← first-week-of-month bias
  ├── compute_stats()            ← win rate, PF, Sharpe, equity curve
  └── return dict                ← trades, stats, candles, filters_info
        │
        ▼
server.py                        ← Flask JSON serialisation
        │
        ▼
static/index.html                ← Chart.js equity curve + TradingView candles
```

### ORB-30 Filter Chain

```
All trading days
    │ Filter 1: Skip if gap > 0.8% (directional continuation, not ORB)
    │ Filter 2: Skip if VIX outside 12–22 (too calm or too volatile)
    │ Filter 3: Skip Tuesdays (lowest ORB success rate historically)
    │ Filter 4: Skip if ORB range < 40 pts (insufficient range = false breakout)
    │ Filter 5: Skip if DTE 0 (expiry pinning distorts breakouts)
    │ Filter 6: Entry only after 9:45 candle close
    │ Filter 7: SL = ORB opposite end (not ATR — research-backed)
    │ Filter 8: Exit at 14:30 regardless
    ▼
609 filtered trades (2015–2022)
Win rate: 51.6% | Profit factor: 1.20 | Sharpe: 1.13
SHORT bias: 88% of total profit
```
